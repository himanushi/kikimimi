# 00006: 設定ページの WiFi SSID を周辺スキャン結果のプルダウンから選べるようにする

## What / Why

セットアップページ(00001)の SSID は手入力のテキストボックスで、スマホでの入力が面倒かつ打ち間違いやすい。デバイスが周辺の WiFi をスキャンし、設定ページで SSID をプルダウンから選択 → パスワードと OpenAI API キーを入力 → 3 つを保存、という流れにする。

## 受け入れ基準

- [ ] スキャン結果の整形(SSID の重複除去・電波強度順ソート・空 SSID 除外)が純関数で分離され、ホストテストが通る(検証: `uvx --from platformio pio test -e native`)
- [ ] ビルドが通る(検証: `uvx --from platformio pio run -e cores3`)
- [ ] 設定ページを開くと周辺の WiFi 一覧がプルダウンに表示される(検証: 実機 + スマホ)
- [ ] プルダウンで選択 + パスワード + API キーで保存 → 再起動 → 選んだ WiFi に接続する(検証: 実機)
- [ ] 一覧に出ない SSID(ステルス等)のための手入力欄も残っている(検証: 実機 + スマホでページ確認)
- [ ] 「再スキャン」でプルダウンが更新できる(検証: 実機 + スマホ)

## やらないこと

- 複数 WiFi の保存(保存は従来どおり 1 件)
- スキャン結果の暗号化方式表示・詳細情報
- 設定ページの見た目の作り込み(素のフォームのまま)

## How(実装方針)

- 依存: issue 00005(同じ portal.cpp を触るため、00005 の完了後に着手)
- SoftAP 動作中のスキャンは `WiFi.mode(WIFI_AP_STA)` + 非同期スキャン(`WiFi.scanNetworks(true)`)で行い、AP を止めない。同期スキャンでブロックしない
- スキャン結果の整形(重複除去・RSSI 降順・空除外)は `src/pure/` の純関数(入力は SSID+RSSI のリスト)
- フォームは `<select>` + 手入力 `<input>` の併設(select で「手入力」を選ぶと input を使う、程度の素朴な作り)。再スキャンはページ再読み込みで新しい結果を反映
