# 00005: AP にクライアントが接続したら設定 URL の案内表示に切り替える

## What / Why

セットアップポータル画面は現在、AP 接続用の QR(00004)を表示したままになる。スマホが AP に接続した後は「ブラウザで http://192.168.4.1 を開く」手順が残っており、IP の手打ちが面倒。

AP への接続台数が 0 → 1 以上になったら、画面を「設定ページを開く案内(URL テキスト + URL の QR コード)」に自動で切り替える。URL の QR をカメラで読めばブラウザが直接開く。接続が 0 台に戻ったら AP 接続用 QR の表示に戻す。

## 受け入れ基準

- [ ] 表示状態の判定(接続台数 → AP 接続案内 / URL 案内)が純関数で分離され、ホストテストが通る(検証: `uvx --from platformio pio test -e native`)
- [ ] ビルドが通る(検証: `uvx --from platformio pio run -e cores3`)
- [ ] スマホが AP に接続すると、画面が URL テキスト + URL QR の表示に切り替わる(検証: 実機 + スマホ)
- [ ] URL QR をカメラで読むとブラウザで設定ページが開く(検証: 実機 + スマホ)
- [ ] スマホが AP から切断されると、AP 接続用 QR の表示に戻る(検証: 実機)

## やらないこと

- captive portal の自動ポップアップ改善(OS 依存の検出 URL 対応は別途)
- ポータル以外の画面遷移の変更

## How(実装方針)

- 依存: issue 00004(ポータル画面の QR レイアウト)
- `WiFi.softAPgetStationNum()` を portal のループでポーリングし、台数が変化したら表示を切り替える(portal.cpp にコールバックまたは状態取得関数を追加し、描画は main.cpp 側)
- URL QR のペイロードは URL 文字列そのまま(`http://192.168.4.1/`)。API キー等の秘密情報は含まれない
- 00004 の「やらないこと(設定 URL 側の QR 化)」は動線が悪いことがユーザー確認されたため本 issue で解除する
