# 00012: デバイス上の設定画面(音量ほか)

## What / Why

現在、ホーム画面の設定ボタンは直接 AP ポータルを起動する。間にデバイス上の「設定画面」を挟み、本体だけで完結する設定(まず音量)と、Web 設定(WiFi・API キー)への導線をまとめる。

設定画面の項目:
- **音量**: スピーカー音量の調整(+/− ボタンまたは段階選択)。変更は即時反映し、確認音を鳴らす。NVS に保存して次回起動時も有効
- **WiFi / API キー設定**: 従来どおりセットアップポータルを起動(00010 以降は STA の URL QR に置き換わる)
- **戻る**: ホーム画面へ

依存: 00009(main.cpp の画面まわり)。00010 はこの画面の「WiFi / API キー設定」項目を置き換える形で後続。

## 受け入れ基準

- [ ] 設定画面のレイアウトとタップ判定が純関数 `src/pure/settings_screen.*` に分離されテストされる(検証: `uvx --from platformio pio test -e native` green。先に red。ケース: 音量 +/− / WiFi 設定 / 戻る / 枠外)
- [ ] 音量値の増減・クランプ(範囲 0〜255 相当を段階化)が純関数でテストされる(検証: native テスト。最小・最大で止まる)
- [ ] ホームの設定ボタン → 設定画面が開き、「戻る」でホームに戻る(検証: 実機)
- [ ] 音量 +/− で即時に確認音が鳴り、音量が変わったことが分かる(検証: 実機)
- [ ] 音量は NVS に保存され、再起動後の対話の再生音量に反映される(検証: 実機で音量を変えて再起動 → 対話)
- [ ] 「WiFi / API キー設定」タップで従来の AP ポータルが起動する(検証: 実機)
- [ ] cores3 ビルドが通る(検証: `uvx --from platformio pio run -e cores3` SUCCESS)

## やらないこと

- 音量以外の本体設定(画面輝度・マイク感度等。必要になったら項目追加)
- Web 設定ページ側の変更(00010/00011)
- ポータル画面のデザイン変更

## How(実装方針)

- `src/pure/settings_screen.h/.cpp`(新規): レイアウト + ヒットテスト(`SettingsTap { None, VolumeUp, VolumeDown, WifiSetup, Back }`)+ 音量段階の増減関数。home_screen.* のスタイルに合わせる
- `src/config_store.*`: 音量の保存・読込を追加(未保存時のデフォルトは M5Unified の既定音量相当)
- `src/main.cpp`: Mode に設定画面の状態を追加(ホーム ⇔ 設定 ⇔ ポータル)。音量変更時 `M5.Speaker.setVolume()` + 短い確認音。対話開始時(realtime_client 側の Speaker.begin 後)にも保存音量を適用する
- デザインはホーム画面(ダーク + ティール)と統一
