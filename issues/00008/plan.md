# 00008: ホーム画面の刷新(対話ボタン・設定ボタン・バッテリー・時計)

## What / Why

現在の CONNECTED 画面はテキスト 1 行(状態 + 金額)だけで、対話開始も「画面のどこでもタップ」になっている。これを「相棒デバイスのホーム画面」らしい見た目に刷新する:

- 大きめの**対話開始ボタン**(中央)— タップで対話開始
- 小さめの**設定ボタン**(隅)— タップでセットアップポータルを起動(WiFi・API キーの再設定)
- **バッテリー残量**(上部)
- **時刻表示**(上部、NTP 同期・JST)
- ダークテーマ + アクセントカラーの「おしゃれ」なデザイン(丸角ボタン等、M5GFX の描画プリミティブで)

対話中の画面は従来どおり状態ラベル + 累計金額を表示する(会話内容の表示は 00009 で別途)。

## 受け入れ基準

- [ ] レイアウト・タップ判定が純関数 `src/pure/home_screen.*` に分離され、native テストがある(検証: `uvx --from platformio pio test -e native` が green。ボタン矩形のヒットテスト: ボタン内座標 → 対話開始 / 設定、外 → 無反応)
- [ ] テストは先に書いて red を確認してから実装する(検証: red の実行出力を result.md に記録)
- [ ] ホーム画面に大きい対話開始ボタン・小さい設定ボタン・バッテリー %・時刻(HH:MM)が描画される(検証: 実機で目視)
- [ ] 対話開始ボタンのタップで従来どおり Realtime 対話が開始し、対話中は画面タップで終了してホーム画面に戻る(検証: 実機)
- [ ] 設定ボタンのタップでセットアップポータル(AP モード + QR 画面)が起動する(検証: 実機)
- [ ] 時刻は NTP(JST)で同期され、分単位で更新される(検証: 実機で目視。NTP 未同期の間は "--:--" などのプレースホルダ)
- [ ] バッテリー残量は M5.Power.getBatteryLevel() を表示し、定期的(30 秒程度)に更新される(検証: 実機で目視)
- [ ] cores3 ビルドが通る(検証: `uvx --from platformio pio run -e cores3` SUCCESS)

## やらないこと

- 会話内容(トランスクリプト)の表示 → 00009
- ポータル画面自体のデザイン変更
- バッテリーアイコンの充電中アニメーション等の凝った演出
- 画面の明るさ・スリープ制御

## How(実装方針)

- `src/pure/home_screen.h/.cpp`(新規): 画面サイズ(320x240)を入力に、ボタン矩形・ラベル位置などのレイアウトを返す純関数と、タッチ座標 → `HomeTap { None, Talk, Settings }` のヒットテスト。native でテスト
- `src/main.cpp`: ホーム画面の描画(M5GFX: `fillRoundRect` 等。ダーク背景 + アクセント色)、タッチを座標つきで取得してヒットテストに通す。時刻は `configTime(9*3600, 0, "ntp.nict.jp", "pool.ntp.org")` + `getLocalTime`。設定ボタン → 既存の `startPortal()` を呼ぶ(WiFi は AP モードに切り替わる)
- 対話中(Idle 以外)は従来の状態表示画面。Idle に戻ったらホーム画面を再描画
- フォントは既存の efontJA_16 系。デザインの細部(配色・角丸半径)は実装者の裁量
- 参考: `knowhow/`、`../yesman/knowhow/` の `m5stack-*`, `pio-*`
